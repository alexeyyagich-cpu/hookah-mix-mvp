# Hookah Torus — End-to-End Test Script

## Цель
Полная проверка всех пользовательских взаимодействий по цепочке для 3 ролей:
- **Гость** (без авторизации) — публичные страницы
- **Управленец** (owner/manager) — дашборд, управление бизнесом
- **Суперадмин** (я) — админ-панель платформы

**Домен**: https://hookahtorus.com
**Демо**: demo@hookahtorus.com / demo2026!
**Локали**: ru / en / de (проверять переключение)

## ФАЗА 9 — Полная цепочка (Happy Path)

### Сценарий: "Один рабочий день кальянной"

1. **Утро**: Открыть смену → начальная касса 200€
2. **Инвентарь**: Проверить остатки → дозаказать через маркетплейс (если < threshold)
3. **Бронь**: Проверить бронирования на сегодня → подтвердить pending
4. **Первый гость**:
   - Бронь → привязать стол → статус "occupied"
   - Создать сессию: 3 табака, 20г, Al Fakher + Adalya + Tangiers
   - Записать гостя → visit_count++, loyalty check
5. **Заказ в бар**: Гость заказывает Mojito через QR → KDS: new → preparing → ready → served
6. **Второй гость (walk-in)**:
   - Стол свободен → occupied
   - Рекомендация микса через recommend engine
   - Сессия создана, инвентарь списан
7. **Смена бартендера**: Новый сотрудник входит по invite → видит только bar + KDS
8. **Вечер**: Ещё 5 сессий, 10 бар-продаж
9. **Закрытие**:
   - Все столы → available/cleaning
   - Закрыть смену → сверка кассы
   - Проверить отчёт: P&L за сегодня
   - Telegram: `/report` → получить сводку
10. **Ночь**: Cron job проверяет low stock → email alert
