# Hookah Torus — Бизнес-план: Германия
**Дата: Февраль 2026**

---

## 1. Резюме (Executive Summary)

**Hookah Torus** — B2B SaaS-платформа для управления кальянными заведениями + собственный табачный бизнес (оптовые поставки табака).

**Стратегия Германии — два направления:**
1. **SaaS**: Hookah Torus как add-on к ready2order POS → standalone Kassensystem
2. **Commerce**: Собственный табачный магазин/оптовый склад → маркетплейс на платформе

**Уникальность**: Ни одна POS-система в Германии не умеет трекать расход табака по граммам, управлять рецептами миксов и считать себестоимость сессий. Ни один табачный магазин не имеет интеграции с системой управления заведением.

### 2.1 Размер рынка
| Метрика | Значение |
|---|---|
| Рынок кальянного табака Германия (OOH) | $9.36M (2025) → $38.44M (2034) |
| CAGR | 19.75% |
| Количество кальянных в Германии | ~6,000-8,000 |
| Ключевые города | Берлин, Кёльн, Рурская область, Гамбург, Франкфурт, Мюнхен |

### 2.2 Проблемы рынка (Pain Points)
1. **Учёт табака** — везде Excel или бумага
2. **Себестоимость** — операторы не знают какие миксы прибыльные
3. **Налоговые проверки** — >50% кальянных проваливают инспекции
4. **Чёрный рынок** — до 80% табака нелегальный, легальные операторы страдают
5. **Нет связи POS ↔ инвентарь** — касса видит "1x Кальян", но не знает какой табак
6. **Заказ табака** — по телефону/email, без автоматизации

## 3. Продукт: SaaS-платформа

### 3.1 Текущее состояние (MVP)
- Учёт инвентаря, калькулятор миксов, сессии, гости, отзывы, бронирования
- План зала, маркетплейс (прототип), управление командой
- Тарифы: Free / Pro ($9/мес) / Enterprise ($29/мес)
- Стек: Next.js, Supabase, Vercel

### 3.2 Интеграция с ready2order POS

**Почему ready2order:**
- REST API + webhooks + PHP/Node.js SDK
- 60 запросов/мин на аккаунт
- Бесплатный API (включён в подписку POS)
- Gastro-пакет с управлением столами
- Популярен в Германии/Австрии

**Архитектура интеграции:**
```
Hookah Torus (Next.js + Supabase)
    │
    ├── OAuth-like подключение (Grant Access Token → Account Token)
    ├── Push: каталог табака → ready2order Products
    ├── Push: столы/зоны → ready2order Tables
    ├── Pull: Webhooks (invoice.created, orderItem.created)
    ├── Sync: инвентарь → Stock batch endpoints
    └── Create: чеки → POST /document/invoice
```

**Ключевые API-эндпоинты:**
| Действие | Эндпоинт |
|---|---|
| Создать продукты пакетно | `POST /v1/product/batch` |
| Обновить остатки | `POST /v1/product/stock/batch` |
| Получить продажи | Webhook `invoice.created` |
| Создать чек | `POST /v1/document/invoice` |
| Печать чека | `POST /v1/document/invoice/{id}/print` |
| Синхронизация клиентов | `GET/POST /v1/customers` |
| Настройка вебхуков | `PUT /v1/webhook` + `PUT /v1/webhook/events` |

**Токен-система:**
1. Developer Token (наш, один на приложение)
2. Grant Access Token (временный, 10 мин)
3. Account Token (постоянный, один на кальянную)

**Что нужно добавить в Supabase:**
```sql
CREATE TABLE r2o_connections (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  profile_id UUID REFERENCES profiles(id),
  account_token TEXT NOT NULL,  -- encrypted
  webhook_secret TEXT,
  connected_at TIMESTAMPTZ DEFAULT now(),
  last_sync_at TIMESTAMPTZ,
  sync_status TEXT DEFAULT 'pending'
);

CREATE TABLE r2o_product_mappings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  profile_id UUID REFERENCES profiles(id),
  local_tobacco_id TEXT NOT NULL,
  r2o_product_id INTEGER NOT NULL,
  r2o_item_number TEXT,
  last_synced_at TIMESTAMPTZ
);
```

## 4. Продукт: Табачный магазин

### 4.1 Бизнес-модель

**B2B-оптовик** → поставки напрямую в кальянные через маркетплейс Hookah Torus.

**Преимущества перед конкурентами (Shisha-World, Moze, Hookain):**
- Интеграция с нашим SaaS = авто-заказ когда табак заканчивается
- Отслеживание расхода в реальном времени
- Персональные рекомендации на основе популярных миксов
- Подписка на регулярные поставки

3. **Tangiers** — нишевый, высокая наценка, лояльная аудитория

### 4.4 Табачный налог (Tabaksteuer) — КРИТИЧНО

Расчёт на 1 кг табака (2025, retail ~100€/кг):
| Компонент | Сумма |
|---|---|
| Базовый налог (фикс) | 15.66€ |
| Базовый налог (13.13% от розницы) | ~13.13€ |
| Доп. налог кальянный табак | 21.00€ (2025) / **23.00€ (2026)** |
| **Итого акциз** | **~49.79€** |
| НДС (19%) | ~19.00€ |
| **Итого государству** | **~68.79€ из 100€** |

> **>50% розничной цены = налоги.** Маржа крайне сжата.

### 4.5 Лицензии и разрешения

**Для розницы (покупка уже отаксованного товара):**

│         └─────────┬─────────────────┘                │
│                   │                                   │
│         ┌─────────▼─────────┐                        │
│         │  МАРКЕТПЛЕЙС      │                        │
│         │                    │                        │
│         │ • Наш магазин      │                        │
│         │ • Другие поставщики│                        │

**Ключевая синергия:**
1. Бар использует SaaS → видит что табак заканчивается → авто-заказ у нас
2. Данные о популярных миксах → мы знаем какой табак заказывать на склад
3. Lock-in: SaaS + поставки = сложно уйти к конкуренту
4. Cross-sell: подписка SaaS + подписка на поставки
